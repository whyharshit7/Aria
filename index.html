The Problem

Cognition Is Not Modular

Modern deep learning treats perception, memory, and reasoning as fundamentally separate problems. The Transformer architecture embodies this fragmentation — and it creates structural limitations that incremental fixes cannot solve.

👁

Perception as Encoding

Encoders map raw input to distributed representations, but have no mechanism to use expectations or predictions to shape what is perceived. Perception is passive, not active.

🧠

Memory as a Window

A fixed-length context window imposes a hard boundary on temporal integration. There's no persistent, mutable memory — no way to learn from interaction without full retraining.

⚡

Reasoning as a Side-Effect

Reasoning emerges implicitly through successive attention layers or is explicitly elicited through chain-of-thought prompting. Depth of reasoning is fixed regardless of problem difficulty.

Biological Precedent

The Brain Unifies Everything

The mammalian neocortex doesn't implement perception, memory, and reasoning as separate subsystems. It implements a hierarchical predictive coding scheme where all three emerge from the same underlying process.

Top-Down

Predictions Flow Downward

Higher cortical areas generate top-down predictions of expected activity in lower areas, creating an internal model of the world.

Bottom-Up

Errors Flow Upward

Lower areas compute prediction errors — the difference between actual and predicted activity — and propagate only the surprises upward.

Multi-Scale

Multiple Timescales

The cortex operates over multiple timescales simultaneously, with intrinsic neural time constants varying systematically across the cortical hierarchy.

Memory

Complementary Systems

Memory is embedded in synaptic weights (semantic), hippocampal traces (episodic), and sustained prefrontal activity (working memory).

Core Principles

Three Axioms

ARIA is built on three foundational design principles inspired by how biological cognition actually works.

01

Prediction Is the Universal Currency

The system actively predicts its inputs at every level of a processing hierarchy, and only the prediction errors — the surprises — drive further computation. This implements the free energy minimization principle as an architectural prior.

02

Memory Is a State, Not a Database

Memory is integral to the system's computational state, manifesting in three complementary forms: fast episodic traces, capacity-limited working memory registers, and slow parametric (semantic) knowledge — with a consolidation process that migrates information between them.

03

Reasoning Is Variable-Depth Recursion

The depth of computation is adaptive, not fixed. Simple inputs that are well-predicted require minimal processing; complex or novel inputs require deep processing. Reasoning is what happens when the recurrent loop runs without external input.

Memory Substrate

Episodic Memory

Fast one-shot storage with importance-weighted forgetting. 65,536 entries with key-value retrieval. Inspired by modern Hopfield networks.

↓ consolidation ↓

Working Memory

7 register slots with gated writes. Only higher levels (≥2) can write, reflecting top-down executive control. Capacity informed by Miller's 7±2.

↓ consolidation ↓

Semantic Memory

The network weights themselves — the slowest-changing store. Updated continuously through consolidation for lifelong learning.

Core Component

Predictive State Column

The fundamental computational unit of ARIA, inspired by the cortical column. Each PSC integrates bottom-up errors, top-down predictions, memory, and working memory through precision-weighted gating.

fl State Transition

A gated recurrent block that integrates all information sources: current state, prediction errors, top-down predictions, memory contents, and working memory.

f(·) = (1 − z) ⊙ s(t) + z ⊙ s̃ − s(t)
// GRU-like gating with multi-source input

gl Downward Prediction

A two-layer MLP that generates predictions of the expected state at the level below. Prediction errors drive all learning.

g(s) = W_g · ϕ(W_g1 · s + b_g1) + b_g
// 2-layer MLP, SiLU activation

πl Precision Estimator

Outputs non-negative precision weights for each information source — controlling how much to trust bottom-up errors, top-down predictions, and memory.

π(s) = Softplus(W_π · s + b_π) ∈ ℝ³⁺
// Three groups: π↑, π↓, πᵐ

ql Memory Query

Projects the current state into a query vector for episodic memory retrieval, enabling context-dependent recall.

q(s) = W_q · s + b_q
// Linear projection to query space
Mathematical Framework

Grounded in Free Energy

ARIA's training objective and dynamics are derived from variational free energy minimization — providing a principled, unified objective.

Generative Model

ARIA defines a hierarchical generative model where each level predicts the level below, with learned precision.

p(x, s₀, …, s_{L-1}) = p(x | s₀) · ∏ p(s_{l-1} | s_l) · p(s_{L-1})

Variational Free Energy

The system minimizes variational free energy — an upper bound on surprise. Under the Laplace approximation, this becomes a precision-weighted sum of prediction errors plus a complexity term.

F ≈ Σ ½ ε_l⊤ Π_l ε_l + ½ Σ log|Π_l⁻¹|

Training Objective

The total loss combines hierarchical prediction errors, complexity regularization, task-specific loss, and adaptive computation cost.

ℒ = Σ λ_l ‖ε_l‖²_{Π_l} + β · D_KL[q‖p] + γ · ℒ_task + δ · ℒ_compute

Adaptive Sparsity Theorem

In a well-trained system, predictable inputs are processed cheaply. For natural language with typical predictability values, this yields ~72% savings over uniform processing.

Cost(t) = Σ (1/τ_l) · [c_base + (1 − ρ_l) · c_error]
Key Mechanism

Adaptive Computation

ARIA allocates computation proportionally to problem difficulty. Simple, well-predicted inputs converge quickly; novel or complex inputs trigger deeper processing.

⚙️ Algorithm — Adaptive Inference
procedure INFER(input_sequence, max_iterations)
  Initialize s_l ← s_l⁰ for all l
  for each input x(t) in input_sequence do
    for iter = 1 to max_iterations do
      for l = 0 to L−1 do
        if t mod τ_l = 0 then
          s_l ← PSC_l.step(s_l, ε_{l-1}, pred_{l+1}, memory, W)
      if Δ(t) < δ_converge and C(t) > δ_confidence then
        break // early exit — input is well-predicted
    if output_requested then
      output ← OutputHead(s₀, …, s_{L-1}, W)

💡 Internal Reasoning Mode

Reasoning is identical to inference except the external input is empty. The recurrent dynamics continue driven by top-down predictions, memory retrievals, and working memory manipulation — reasoning as extended recurrence.

Advantages

Why This Matters

01

Unified Framework

Perception, memory, and reasoning arise from a single computational principle — avoiding the engineering complexity of modular systems.

02

Adaptive Efficiency

Prediction error propagation provides natural adaptive computation without a separate halting mechanism. Predictable inputs are cheap.

03

Continuous Learning

Architecturally integrated consolidation addresses the inability of current systems to learn from deployment-time interactions.

04

Uncertainty Awareness

Precision weighting provides native uncertainty estimation, potentially reducing hallucination and enabling calibrated confidence.

The question is not whether prediction, memory, and reasoning should be unified — the brain demonstrates that they can be.

The question is whether we can build artificial systems that realize this unification efficiently and at scale. ARIA is our attempt to sketch what such a system might look like.
